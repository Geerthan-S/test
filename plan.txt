You are a senior Next.js + TypeScript + Tailwind CSS architect working on the Dockside Constructions website. Your task is to completely refactor the existing Projects module from a dedicated case-study page architecture to a modal/popup architecture.

# OBJECTIVE

The current implementation opens a dedicated page when a user clicks "View Project". The client has decided that project details should no longer exist as separate pages. Instead, all project details must open inside a professional full-screen popup/modal.

Your task is to perform a complete architectural migration.

---

Delete every component, route, style, helper, and API implementation that exists only for project case-study pages.

  * page.tsx files

  * metadata generators
  * project detail components
  * project detail CSS
  * project detail SCSS
  * project detail hooks
  * project detail utilities
  * project detail API handlers
  * project detail schemas
  * project detail types that are no longer needed

* Remove all imports and references to:

  * challenge sections
  * solution sections
  * execution flow sections
  * related services sections
  * proof points sections
  * project dossier sections
  * CTA sections
  * timeline case-study components
  * case-study SEO generators

* Remove all unused assets related only to project detail pages.

After deletion, ensure:

* no dead code exists

* the application builds successfully

# PART 2 — CONVERT "VIEW PROJECT" TO OPEN A MODAL

Currently:

```tsx
<Link href={`/projects/${project.slug}`}>
    View Project
</Link>
```

Replace this behavior with:

```tsx
<Button onClick={() => openProject(project)}>
    View Project
</Button>
```

Requirements:

* Clicking any project card opens a full-screen modal.
* No page navigation should occur.
* URL should remain unchanged.
* Modal should support ESC close.
* Modal should support backdrop click close.
* Modal should support mobile swipe scrolling.
* Body scroll should lock while modal is open.
